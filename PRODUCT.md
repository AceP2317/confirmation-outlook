# Product

## Register

product

## Platform

web

## Users

Primary: recruiters, hiring engineers, and prospective clients arriving from a portfolio link with about sixty seconds of attention and no supply-chain background — first-time viewers who must be able to orient without a manual. Secondary: the simulated persona the tool demonstrates, a supply-chain operations lead working an at-risk order book who needs to know which materials to chase this week.

## Product Purpose

A public, full-stack demonstration of a predictive order-confirmation tool for the fictional Northpoint Manufacturing, running on synthetic data with a real Python engine behind it (empirical risk model, held-out-week validation, REST + MCP + AI chat). Success: a viewer trusts the engineering depth within a minute, and anyone skeptical can verify any displayed number against the open API and the validation gate.

## Positioning

A predictive tool that shows its work — every probability is a measured frequency, validated on a week the model never saw.

## Brand Personality

Precise, honest, operational. The register of a flight instrument: calm, dense, zero hype. Numbers lead; adjectives are rationed.

## Anti-references

The generic navy-and-blue SaaS analytics dashboard. AI-hype marketing surfaces with glow and gradients. Toy demos whose numbers are decorative rather than computed.

## Design Principles

- Instrument, not brochure: the surface reads like equipment an operator relies on.
- Every number inspectable: values trace to an API the viewer can open.
- Honest surfaces: caveats (memoryless headline, volume dispersion, synthetic data) are displayed, never hidden.
- Legible on first contact: plain-language intros with live numbers, glossary tooltips on jargon.
- Density with hierarchy: dense is fine; unranked is not.

## Accessibility & Inclusion

WCAG AA contrast throughout (body 4.5:1). Full reduced-motion alternative. Keyboard-visible focus states. Color never the only carrier of risk state (icons/text accompany).
